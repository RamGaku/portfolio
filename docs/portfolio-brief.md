# Portfolio Brief

> 공개 가능한 경력 KB 기준 요약본입니다. 원본 Obsidian 노트의 계정, IP, 접속 정보, 고객 내부 정보는 포함하지 않습니다.

## Core Positioning

**한 줄 소개**

현장의 모호한 운영 문제를 데이터 구조, 워크플로우, 화면, 고객 산출물로 번역하는 Forward Deployed Engineer형 엔지니어.

**지원 포지셔닝**

NADA에서 산업 장비, 통신 프로토콜, Edge 데이터 플랫폼, 고객 산출물을 하나의 흐름으로 연결해 왔습니다. 고객 현장에서 요구사항을 파악하고, 장비 통신과 데이터 수집 구조를 맞추며, 운영자가 실제로 확인할 수 있는 화면, API, 문서까지 이어 붙이는 역할을 수행했습니다.

Enhans의 Forward Deployed Engineer 역할에는 고객 문제를 빠르게 정의하고, Commerce OS의 Ontology, Workflow, Agent 같은 레이어를 고객 맞춤 시스템으로 구현하는 능력이 중요합니다. 저는 직접 Commerce OS를 개발했다고 말하지 않습니다. 대신 태그/주소/채널을 운영 객체처럼 정렬한 경험, 배포 점검과 mapping 자동화를 workflow로 만든 경험, API/DB/화면/문서까지 연결한 경험을 Enhans FDE의 인접 근거로 제시합니다.

## Fit To Enhans

### 1. 고객 문제 정의

현장 요구사항을 단순 요청 목록으로 받아들이지 않고, 장비 통신, 데이터 파이프라인, UI/리포트, 운영 문서로 번역했습니다.

### 2. End-to-end 구현

고객사 환경에 맞춰 OPC, Modbus, REST, RTDB, Grafana, Edge Server 구성을 조정했습니다. 데이터가 수집되는 지점부터 사용자가 확인하는 화면까지 전체 흐름을 검증했습니다.

### 3. 빠른 문제 해결

문제가 재현되지 않는 상황에서도 로그, 프로토콜, 타임아웃, DB 흐름을 추적해 원인을 좁혔습니다. 단순 개발보다 현장 디버깅과 운영 검증을 함께 수행했습니다.

### 4. 제품화 감각

설치 후 동작 확인, 서비스 상태 점검, 설정 자동화, 고객 문서화까지 진행해 일회성 대응이 아니라 재사용 가능한 제품 흐름으로 정리했습니다.

## Chatbot Concept

**Purpose**

포트폴리오 방문자가 "이 사람이 어떤 일을 했는지"를 직접 질문하면, 공개 경력 KB에서 근거를 찾아 답하고 관련 프로젝트 섹션으로 이동할 수 있게 합니다.

**Default Message**

공개 경력 KB 기준으로 답합니다. Vertex AI와 공개 KB를 연결해 Enhans 적합성, 프로젝트별 데이터 플로우, 경주풍력 Digital Twin, Modbus Mapping Skill, 폐쇄망 OnlineTSI, 산업용 인터페이스, Edge Computing, 펌웨어 디버깅 경험을 물어볼 수 있습니다.

**Suggested Prompts**

- Commerce OS Ontology Workflow Agent와 맞닿는 근거를 보여줘
- Enhans FDE 역할에 맞는 근거를 프로젝트별로 보여줘
- 프로젝트별 데이터 플로우와 레이어 스택을 설명해줘
- 경주풍력 Digital Twin 프로젝트로 이동하고 싶어
- 폐쇄망 OnlineTSI와 Edge Computing 경험 차이를 설명해줘
- Modbus Mapping Skill 프로젝트를 설명해줘
- 펌웨어/프로토콜 디버깅 경험을 설명해줘

**Safety Rule**

챗봇은 공개 가능한 경력 정보만 답합니다. 계정, 비밀번호, IP, VPN, 원격 접속 정보, 고객 내부 정보는 답변하지 않습니다. 다만 "원격 조회 문제 해결"처럼 공개 가능한 경력 질문은 차단하지 않아야 합니다.

## Deployment Direction

**Recommended Stack**

- Firebase Hosting: 정적 포트폴리오, 커스텀 도메인, SSL
- Cloud Run: `/api/chat` 서버, Vertex AI 인증 정보 보관
- Vertex AI: 공개 KB 기반 답변 생성
- Firestore Vector Search 또는 Vertex AI Search: 공개 KB 인덱스 확장

**Deployment Copy**

정적 프론트엔드는 Firebase Hosting에 올리고, Vertex AI를 호출하는 API는 Cloud Run으로 분리합니다. 커스텀 도메인과 SSL은 Hosting에서 처리하고, 공개 KB만 RAG 인덱스로 확장합니다.
