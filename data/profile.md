## 솔루션: VNET-7000
- 개요: Server-Client 구조로 현장에 설치되어 회전설비의 진동을 감시하는 솔루션. 실시간 알람, 데이터 수집, 진동 데이터 외부 송신, 분석 기능 제공.
- 기술스택: C++(펌웨어), .NET Framework, WinForm (Server / Monitor / DataServer), MSSQL DB
- 내가 한 일: 유지보수 및 요구 기능 추가, 외부 데이터 인터페이스

## 솔루션: VNET-7300
- 개요: 포터블 형태의 진동 분석기. 휴대·이동하여 타겟 설비의 진동을 현장에서 측정·분석한다.
- 내가 한 일: 유지보수 및 요구 기능 추가

### VNET-7000 / 7300 — 배운 점 · 힘들었던 점
- 배운 점
  - 최초의 센서부터 응용프로그램까지 모든 레이어를 경험함
  - 외부 협력사와 데이터를 주고받는 경험
  - 진동 신호는 굉장히 큰 데이터이며, 이를 다룬 경험
- 힘들었던 점
  - 고객사가 대부분 발전사이며 굉장히 보수적
  - 폐쇄망에서 운영되어 물리적 방문 및 조치가 어려울 수 있음
  - 임베디드(펌웨어) 환경의 개발 및 기능 변경 난이도

## 솔루션: 경주풍력 Digital Twin
- 개요: 경주풍력 2발전소의 SCADA 시스템에 진동 데이터 취득을 결합한 시스템. 각 Edge 디바이스에서 데이터를 처리하고 Viewer로 실시간 3D 모델링과 연결해 Digital Twin 과제 수행.
- 기술스택: .NET(Edge Server), WPF(Dt Viewer), RTDB
- 내가 한 일: 전체 솔루션 개발 참여
- 배운 점
  - 개발 기술 선정의 중요성 (WPF + Ab3d 엔진을 사용해 기한 내 완수)
  - Fast Prototype의 중요성 (빠른 MVP 결과물 시연으로 고객 만족)
- 힘들었던 점
  - 시간 부족 (약 6개월)
  - 현장(풍력발전기 내부)의 자유로운 방문이 어려움

## 프로젝트: 청송양수 고진동 (실패에서 배운 것)
- 담당: 청송양수발전소 고진동 발생 원인 분석 프로젝트에서 진동 데이터 측정 및 Export 부문
- 내용: 기존 현장에서 돌던 VNET-7000에 기능을 추가해 진동 데이터를 중앙연구원으로 전달하는 프로젝트
- 문제점
  1. '진동' 카테고리였기 때문에 VNET-7000으로 하면 되는 것으로 합의되어 Top-Down으로 기획·계약
  2. 진동 발생 부위가 '회전체'가 아니었음 → 여기서부터 프로젝트 설계에 괴리 발생
  3. 이미 납품하기로 된 하드웨어·소프트웨어가 계약되어 있어, 용도에 맞지 않는 플랫폼에 기능을 추가
  4. 용도에 맞지 않다 보니 가공된 데이터의 품질과 사용성이 크게 저하
  5. 요구 스펙상 완료는 됐지만 결과적으로 실패한 프로젝트
- 아쉬운 점
  1. 진동 도메인을 잘 알지 못한 채로 프로젝트가 설계된 점
  2. 최종 사용 고객의 정확한 요구사항을 구체적으로 추출하지 못함
  3. 타 부서와의 대화 부족
- 배운 점
  1. 같은 도메인이라도 직군·개인별로 이해도 차이가 크다. 관련이 있다면 설계 단계부터 꼭 관여해야 더 큰 사고를 막는다.
  2. 최종 사용자의 정확하고 구체적인 요구사항 추출이 필요하다. 사용자 본인도 추상적으로만 생각하는 경우가 있어, 빠른 MVP로 생각한 게 맞는지 확인시켜줘야 한다.
  3. 사람과의 관계의 중요성. 현장 실무자들과 좋은 관계를 유지한 덕에 곤란한 상황에서도 편의를 봐주는 경우가 종종 있었고, 실제 현장 일에 큰 도움이 됐다.

## 프로젝트: VNET-7000 네트워크 이중화
- 내용: VNET-7000 솔루션의 이중화 수행
- 문제점
  - '이중화 범위'에 대한 정의 부재. 센서-측정장비-서버-DB-모니터까지 이어지는데 어디부터 어디까지 이중화할지 불명확
  - 이미 개발 완료된 상용 제품에 이중화 솔루션을 적용해야 하는 문제
- 해결 방법
  - Cisco HSRP(Hot Standby Router Protocol)로 네트워크 이중화 구현
  - MSSQL Failover Cluster로 DB 이중화 구현
- 아쉬운 점
  1. 솔루션 자체에 기능으로 녹이지 못한 점 (녹였다면 동일 요구에 재대응 가능)
  2. 타사 제품 기능을 각각 이용해 전체 솔루션이 너무 복잡해짐
- 배운 점
  1. 기술 용어의 모호함의 함정. '이중화'는 솔루션을 2벌 깔지 않는 이상 어딘가에서 합의가 필요하며, 이를 모두가 알아야 한다.
  2. 상용 제품 솔루션들을 조합해 요구 기능을 만들어내는 것 — 방법은 언제나 존재한다.
